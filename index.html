Projects



UI/UX Design

As a part of coursework for Human Machine Interaction my task was to design user interface for mobile application named Basket Daily following good HCI practices.
Basket Daily is a mobile application for online grocery delivery service. It primarily delivers grocery goods found in convenience stores, home essentials and food supplies to its customers. The application provides an easy relaxed way of browsing and shopping for groceries, enabling users to select and buy products of their choice within the comfort of their home or office. The user interface is simple, easy to use and navigate and therefore is intuitive and consistent.






Restaurant Management System

A restaurant management system is a widely required system used to ensure efficient and smooth functioning of a restaurant. It is required to keep track of the diners checking in, each tables order and ensuring the bill is totalled correctly. The program is written in C and can perform following funnction-

  • Reservation and accept details of new diner
  • If required table is available, allot it else keep the diner in the waiting list
  • Display menu and accept order from diner
  • Send order to kitchen
  • Once a diner wishes to pay bill, the total bill is shown
  • Waiting list is analysed to check whether any diners waiting to occupy the table

Different data structures are used in each portion of the program to ensure that the data is organised efficiently and be able to access/retrieve it efficiently whenever required. Graphics have been used in the program to provide a simple user interface for a waiter to operate.

TourTime: Virtual Tourist Guide

This project involves a cross platform mobile application using which user can click a picture of a Landmark or Tourist Destination they are visiting. User-provided image is processed by a Convolutional Neural Network in the Cloud to identify the landmark and then return information about it, as well as other nearby locations. Users can also suggest relevant information about a landmark which will be verified using a crowd review system. A Guided Tour Itinerary will be presented to the user based on frequent tourist visitation patterns.

Technologies used: Python Web Scraper, Ionic, NestJS Central Web Server, Dashboard and the AI module, Flask & TensorFlow Module to process images and identify landmarks using a Neural Network, VueJS Admin Dashboard, Docker, SQLite






Scrivener

Scrivener is a video transcript summarizer for Youtube videos which accepts a youtube URL link, collects the caption at every sentence and then provides the summary of the complete video. As a part of coursework for Software Engineering our task was to significantly improve and add additional features to this already developed project. As part of team we made following contributions-

  • Enhanced product quality by improving the summarization model.
  • Greatly improved summary formatting to improve readability
  • Provided Sentiment Analysis of the generated summary
  • Improved Heroku deployment
